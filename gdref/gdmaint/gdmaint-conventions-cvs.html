CVS Suggestions

The source in CVS should always work correctly. Those who break the build will be tickled with feathers or suffer a similar fate. Always, always, always test your code before committing it. For compiler changes, you should at least run the test suites in gd/src/tests as a smoke test. Also, a full bootstrap cycle is strongly recommended before commiting more complicated changes.
